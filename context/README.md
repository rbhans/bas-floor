# Project Context Index

Read this file first.

## Always relevant
- `PROJECT_OVERVIEW.md` - what this project is, who it is for, and the main goals.
- `CURRENT_STATE.md` - what exists now, what is in progress, and what should happen next.

## Conditional
- `ARCHITECTURE.md` - read for structural, backend, system, data-model, or integration changes.
- `DECISIONS.md` - read before changing established patterns.
- `OPEN_QUESTIONS.md` - read when requirements are unclear.
- `EXTERNAL_REFERENCES.md` - read when research, Obsidian notes, PDFs, screenshots, URLs, or outside docs matter.

Keep each file concise. Prefer summaries and links over long explanations. Repo-local Markdown remains the implementation source of truth.
